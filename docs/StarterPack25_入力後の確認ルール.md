# FarmPro 入力後の確認ルール

## 目的

入力したデータが正しく保存されているか、あとで見つけやすいかを確認します。

---

# 1. 登録直後の確認

データを登録したら、必ず一覧画面に戻って確認します。

確認すること:

- 一覧に表示されているか
- 日付が正しいか
- 対象牛・子牛が正しいか
- 状態が正しいか

---

# 2. 編集後の確認

編集したら、もう一度その行を確認します。

確認すること:

- 編集した項目が変わっているか
- 変えていない項目が消えていないか
- メモが残っているか

---

# 3. 削除前の確認

削除は慎重に行います。

確認すること:

- 本当に削除する記録か
- 別の牛の記録ではないか
- バックアップがあるか

削除前に迷ったら、削除せずメモで対応します。

---

# 4. 予定完了時の確認

作業が終わった予定は「完了」にします。

確認すること:

- 完了にする予定が正しいか
- 作業内容を治療・ワクチン・繁殖などに記録したか
- 次回予定が必要なら登録したか

---

# 5. 治療入力後の確認

治療記録は重要です。

確認すること:

- 対象牛が正しい
- 治療日が正しい
- 薬剤名が入っている
- 投薬量が入っている
- 休薬終了日が正しい
- 経過が正しい

---

# 6. ワクチン入力後の確認

確認すること:

- 対象牛または子牛が正しい
- ワクチン名が正しい
- 接種日が正しい
- 次回予定日が正しい
- 接種済みなら状態が更新されている

---

# 7. 繁殖入力後の確認

確認すること:

- 対象牛が正しい
- 授精日が正しい
- 種雄牛名の表記がいつもと同じ
- 分娩予定日が不自然でない
- 妊娠結果が正しい

---

# 8. バックアップ前の確認

バックアップ前に確認すること:

- 入力したデータが一覧に出ている
- 保存エラーが出ていない
- serverが起動している
- Chromeで最新表示になっている

---

# 9. GitHub保存前の確認

GitHub保存前に確認すること:

- アプリが白い画面ではない
- serverに赤いエラーがない
- clientに赤いエラーがない
- 主要画面が開ける
- `git status` の内容が想定通り

問題なければ保存します。

```bash
git add .
git commit -m "Update farm data"
git push
```

---

# 10. 入力後チェックリスト

- 一覧に表示された
- 対象牛・子牛が正しい
- 日付が正しい
- 状態が正しい
- メモが必要なら入っている
- 次回予定が必要なら入っている
- バックアップした
- 必要ならGitHub保存した
