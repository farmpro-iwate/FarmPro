# Starter Pack 101 対応記録期限アラート 基本設計

## 1. 目的

給与アラート対応記録では、以下を記録できるようになりました。

```text
対応日
子牛名
アラート種別
対応内容
状態
次回確認日
メモ
```

次に必要なのは、対応記録を「登録して終わり」にしないことです。

特に以下の記録を見つけやすくします。

```text
未対応のまま残っている
対応中のまま止まっている
様子見の次回確認日が近い
次回確認日を過ぎている
再確認必要になっている
```

これにより、確認漏れや対応忘れを防ぎます。

---

## 2. 判定ルール案

## 2-1. 期限切れ

次回確認日が今日より前で、状態が `対応済み` ではない場合。

```text
nextCheckDate < 今日
status != 対応済み
```

表示名:

```text
期限切れ
```

優先度:

```text
高
```

---

## 2-2. 今日確認

次回確認日が今日で、状態が `対応済み` ではない場合。

```text
nextCheckDate = 今日
status != 対応済み
```

表示名:

```text
今日確認
```

優先度:

```text
高
```

---

## 2-3. まもなく確認

次回確認日が明日から3日以内で、状態が `対応済み` ではない場合。

```text
今日 < nextCheckDate <= 今日+3日
status != 対応済み
```

表示名:

```text
まもなく確認
```

優先度:

```text
中
```

---

## 2-4. 再確認必要

状態が `再確認必要` の場合。

```text
status = 再確認必要
```

表示名:

```text
再確認必要
```

優先度:

```text
高
```

---

## 2-5. 未対応

状態が `未対応` の場合。

```text
status = 未対応
```

表示名:

```text
未対応
```

優先度:

```text
中
```

## 9. 運用イメージ

実際の使い方は以下です。

```text
1. ホームで給与アラートを見る
2. 対応記録を登録する
3. 次回確認日を入れる
4. ホームに期限アラートが出る
5. 期限が来たら確認する
6. 状態を対応済みまたは様子見に更新する
```

これにより、給与調整後の確認漏れを防ぎます。
