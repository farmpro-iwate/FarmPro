# Starter Pack 148 全体の番号ルール整理

## 1. 目的

FarmPro全体で、牛の番号の呼び方を整理します。

これまで一部画面で、

```text
ID
母牛ID
子牛ID
```

という表示がありました。

ただし、現場で見る番号としては分かりにくいので、今後は以下のルールにします。

```text
耳標番号：画面で主に表示する番号
個体識別番号：正式な管理番号
ID：システム内部用
```

---

## 2. 基本ルール

### 画面で主に表示するもの

```text
耳標番号
```

理由:

```text
現場で牛を見て確認しやすい
作業中にすぐ分かる
入力しやすい
```

---

### 正式な管理番号

```text
個体識別番号
```

理由:

```text
公的・正式な管理で必要
販売・出荷・台帳管理で必要になる場合がある
耳標番号とは別で管理した方が分かりやすい
```

---

### 画面に出さない方がよいもの

```text
ID
```

理由:

```text
システム内部番号に見える
現場では意味が分かりにくい
耳標番号や個体識別番号と混乱する
```

---

## 3. 今後の画面表示ルール

今後の画面では、なるべく以下の表記に統一します。

```text
母牛耳標番号
子牛耳標番号
個体識別番号
```

避けたい表示:

```text
母牛ID
子牛ID
牛ID
calfId
cowId
```

---

## 4. 既存データとの関係

内部データでは、今まで作った都合で以下の名前が残っています。

```text
cowId
calfId
calfName
```

ただし、すぐに内部名を全部変えると壊れやすいです。

そのため、しばらくは以下の方針にします。

```text
内部名はそのまま
画面表示だけ耳標番号に変える
```

例:

```text
内部: cowId
画面: 母牛耳標番号
```

```text
内部: calfName
画面: 子牛耳標番号
```

---

## 5. 分娩記録でのルール

分娩記録では、Pack147から以下の表示にしました。

```text
母牛耳標番号
子牛耳標番号
```

子牛台帳へ登録するときも、子牛耳標番号を中心に確認します。

---

## 6. 牛台帳での理想形

今後、牛台帳では以下の項目があると分かりやすいです。

```text
耳標番号
個体識別番号
名前
生年月日
性別
状態
```

画面で一番目立たせるのは、

```text
耳標番号
```

です。

---

## 7. 子牛台帳での理想形

今後、子牛台帳では以下の項目があると分かりやすいです。

```text
耳標番号
個体識別番号
母牛耳標番号
母牛名
生年月日
性別
出生体重
状態
```

画面で一番目立たせるのは、

```text
耳標番号
```

です。

---

## 8. 出荷・販売での理想形

出荷・販売では、正式な確認が必要になるため、以下の両方があると安心です。

```text
耳標番号
個体識別番号
```

現場で探すときは耳標番号。

正式な記録として残すときは個体識別番号。

この使い分けが自然です。

---

## 9. 変更の進め方

一気に全部変えると危険なので、以下の順番で進めます。

```text
1. 番号ルールをdocsで固定
2. 分娩記録の表示を耳標番号に統一
3. 子牛台帳の表示を耳標番号中心に整理
4. 牛台帳の表示を耳標番号中心に整理
5. 出荷・販売やレポートの表示を整理
```

---

## 10. 結論

FarmProでは今後、以下の考え方で進めます。

```text
現場で見る番号 = 耳標番号
正式な番号 = 個体識別番号
システム内部用 = ID
```

画面では、IDという言葉をできるだけ出さず、耳標番号を中心に表示します。
