# FarmPro 毎日の作業手順

## 目的

毎日の確認・入力・保存の流れを決めて、入力漏れや確認漏れを減らします。

---

# 朝の確認

## 1. アプリを起動する

server と client を起動して、Chromeで開きます。

```text
http://localhost:5173
```

## 2. ホーム画面を確認する

ホーム画面で全体の状況を確認します。

確認する項目:

- 登録牛数
- 子牛数
- 未完了予定
- 治療中
- BLV陽性数
- 近日予定

## 3. アラートを確認する

上部メニューの「アラート」を開きます。

優先して確認するもの:

- 期限切れ予定
- 分娩予定
- ワクチン予定
- BLV次回検査
- 治療中
- 要再診
- 休薬期間中

## 4. カレンダーを確認する

上部メニューの「カレンダー」を開きます。

今日と今週の予定を確認します。

確認するもの:

- 分娩予定
- 妊娠鑑定
- ワクチン
- BLV検査
- 治療
- その他の予定

---

# 作業中の入力

## 1. 治療した場合

「治療管理」に入力します。

最低限入れるもの:

- 対象牛
- 治療日
- 症状
- 薬剤
- 投薬量
- 休薬終了日
- 経過

## 2. ワクチン接種した場合

「ワクチン管理」に入力します。

最低限入れるもの:

- 対象牛または子牛
- ワクチン名
- 接種日
- 次回予定日
- 状態

## 3. 繁殖作業をした場合

「繁殖管理」に入力します。

最低限入れるもの:

- 対象牛
- 発情日
- 授精日
- 種雄牛
- 妊娠結果
- 分娩予定日

## 4. 子牛が生まれた場合

「子牛管理」に入力します。

最低限入れるもの:

- 子牛番号
- 子牛名
- 性別
- 生年月日
- 母牛
- 出生体重

## 5. 新しい予定ができた場合

「予定管理」に入力します。

最低限入れるもの:

- 予定区分
- タイトル
- 対象牛または子牛
- 予定日
- 状態

---

# 作業後の確認

## 1. 入力した画面をもう一度見る

登録した内容が一覧に出ているか確認します。

## 2. 予定を完了にする

終わった予定は「完了」に変更します。

未完了のままだと、アラートに残ります。

## 3. アラートを再確認する

作業後に「アラート」を開いて、残っている注意点を確認します。

## 4. バックアップする

大きな入力をした日は、バックアップ画面から全データバックアップを保存します。

## 5. GitHub保存する

Starter Packを反映した日や、大きくデータを入力した日は GitHub に保存します。

```bash
cd C:\Users\関口\Downloads\FarmPro_StarterPack1\FarmPro_StarterPack1
git status
git add .
git commit -m "Update farm data"
git push
```

# 1日の終わり

## 確認リスト

- 今日の治療記録を入力した
- 今日のワクチン記録を入力した
- 今日の繁殖記録を入力した
- 今日の予定を完了にした
- 明日以降の予定を確認した
- バックアップした
- 必要ならGitHub保存した

## 注意

白い画面やエラーが出ている状態では GitHub 保存しないでください。
まず復旧してから保存します。
