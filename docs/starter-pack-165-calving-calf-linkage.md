# Starter Pack 165: 分娩記録と子牛登録の紐づけ方針

## 目的

子牛台帳・子牛カルテの見やすさ改善が進んだため、次は繁殖・分娩側から自然に子牛登録へつながる導線を整理する。

現場で迷いやすい点は、分娩記録、母牛、子牛登録が別々に見えてしまうこと。
このStarter Packでは、すぐに大きな実装を入れず、画面とデータのつなぎ方を先に決める。

## 基本方針

- 現場表示は耳標番号を中心にする。
- 内部保存はIDを使ってもよいが、画面では耳標番号と名号で確認できるようにする。
- 分娩記録から子牛を登録できる導線を作る。
- 子牛登録後、分娩記録に子牛IDまたは子牛識別情報を戻して紐づける。
- 複雑な自動推測より、確認しやすい候補表示を優先する。
- 既存の子牛台帳・子牛カルテを壊さないように段階的に進める。

## 想定する画面導線

### 1. 分娩記録一覧

分娩記録一覧では、母牛と分娩結果を見ながら、子牛登録状況を確認できるようにする。

表示候補:

- 母牛耳標番号
- 母牛名号
- 分娩予定日
- 実分娩日
- 分娩結果
- 子牛登録状況
- 子牛耳標番号
- 子牛カルテへのリンク

子牛が未登録の場合は、`子牛を登録` ボタンを表示する。

### 2. 分娩記録詳細

分娩記録詳細では、分娩内容と子牛情報を同じ画面で確認できるようにする。

表示候補:

- 母牛情報
- 分娩情報
- 初乳確認
- 出生体重
- 子牛登録状況
- 子牛登録ボタン
- 既存子牛との紐づけ候補

### 3. 子牛登録フォーム

分娩記録から子牛登録へ進む場合、次の値を初期入力する。

- 母牛名または母牛耳標番号
- 生年月日 = 実分娩日
- 出生体重 = 分娩記録の出生体重
- 性別 = 分娩記録の性別
- 備考 = 分娩記録から作成したことが分かるメモ

耳標番号と名号は現場で確認しながら入力する。

### 4. 子牛登録後

子牛登録後は、次のどちらかを選べるようにする。

- 子牛カルテへ移動
- 分娩記録へ戻る

分娩記録から登録した場合は、保存後に分娩記録へ戻る導線があると現場で分かりやすい。

## データ紐づけ方針

### 分娩記録側に持たせたい情報

最低限:

- `motherId` または母牛識別情報
- `calfId`
- `calfNumber`
- `calfName`

可能なら:

- `linkedAt`
- `linkSource`

`calfId` があれば内部リンクとして使いやすい。
`calfNumber` と `calfName` は画面確認用に持っておくと分かりやすい。

### 子牛側に持たせたい情報

最低限:

- `motherName`
- `birthday`
- `sex`
- `startWeight`

可能なら:

- `motherId`
- `calvingRecordId`

`calvingRecordId` があれば、子牛カルテから分娩記録へ戻れる。

## 紐づけ候補の考え方

自動で確定するのではなく、候補表示に留める。

候補条件:

- 母牛が一致する
- 子牛の生年月日が実分娩日に近い
- 性別が一致する
- 出生体重が近い
- 備考に分娩記録由来のメモがある

候補表示では、`この子牛と紐づける` ボタンを用意する。

## 注意点

- いきなり自動紐づけしない。
- 画面表示は耳標番号を優先する。
- IDだけを表示しない。
- 子牛登録フォームに余計な入力項目を増やしすぎない。
- 分娩記録、繁殖記録、子牛台帳の責務を混ぜすぎない。
