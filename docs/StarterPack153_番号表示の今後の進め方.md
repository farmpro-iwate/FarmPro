# Starter Pack 153 番号表示の今後の進め方

## 1. 基本方針

番号表示は、今後も以下の考え方で進めます。

```text
画面で主に見る番号 = 耳標番号
正式な番号 = 個体識別番号
システム内部用 = ID
```

---

## 2. 今やっていること

今は、画面上の言葉を整理しています。

```text
牛ID → 耳標番号
子牛ID → 子牛耳標番号
母牛ID → 母牛耳標番号
個体番号 → 耳標番号 または 個体識別番号
```

---

## 3. まだやらないこと

内部データ名は、すぐには変えません。

```text
id
cowId
calfId
cattleId
```

これらは、プログラム内部では残してOKです。

---

## 4. なぜ時間がかかるか

番号表示は、いろいろな画面にまたがっています。

```text
牛台帳
子牛台帳
分娩記録
繁殖管理
治療管理
ワクチン管理
BLV管理
出荷・販売
レポート
印刷
```

一気に変えると、以下の問題が起きやすいです。

```text
白画面
保存できない
検索できない
別画面との連携が切れる
```

そのため、少しずつ進めるのが安全です。

---

## 5. 今後の優先順位

次に直すなら、以下の順番が安全です。

```text
1. 画面に残った古い表示だけを置き換える
2. 牛台帳の登録・編集画面を見直す
3. 子牛台帳の登録・編集画面を見直す
4. 印刷・レポート表示を整理する
5. 個体識別番号の専用欄を追加するか検討する
```

---

## 6. 個体識別番号欄について

今後、本格的に整えるなら、以下を分けるのが理想です。

```text
耳標番号
個体識別番号
名前
```

ただし、これはデータ構造に関わるため、後で安全に進めます。

---

## 7. 結論

今は、画面表示だけ耳標番号中心に寄せる段階です。

データの中身を大きく変えるのは後回しにします。
