# FarmPro 動作確認手順

FarmPro を叩き台として動かし、使用感から次の改善点を決めるための手順です。

## 目的

- 主要画面が開けるか確認する
- 白画面やエラーが出ないか確認する
- 実際の繁殖農家の作業順で使えるか確認する
- スマホで使いにくい場所を見つける
- 次に直す場所を決める

## 事前準備

- Windows PC
- VS Code
- コマンドプロンプト、または VS Code のターミナル
- FarmPro のフォルダ
- ブラウザ

## 1. server を起動する

別のターミナルを開き、server フォルダへ移動します。

```bash
cd server
npm install
npm run dev
```

以下のような表示が出れば、server 側は起動しています。

```text
http://localhost:4000
```

API確認：

```text
http://localhost:4000/api/health
```

## 2. client を起動する

もう1つターミナルを開き、client フォルダへ移動します。

```bash
cd client
npm install
npm run dev
```

通常は以下で開きます。

```text
http://localhost:5173
```

## 3. 最初に確認すること

- [ ] ブラウザで FarmPro が開く
- [ ] 白画面にならない
- [ ] 上部に「繁殖Farm Pro」と表示される
- [ ] 上部ナビが表示される
- [ ] ホーム画面が表示される

## 5. 実作業の流れで確認する

実際の作業を想定して、以下の順番で確認します。

1. 牛台帳を開く
2. 牛を登録、または既存の牛を見る
3. 耳標番号で牛を見分けられるか確認する
4. 繁殖管理を開く
5. 繁殖記録を登録、または確認する
6. 分娩予定日を確認する
7. 子牛管理を開く
8. 子牛の日齢、体重、DGを確認する
9. ワクチン、BLV、治療の記録を確認する
10. 出荷販売、経費、月別収支を確認する

## 6. スマホで確認する

PCで開けたあと、iPhone のブラウザでも確認します。

- [ ] 画面が開く
- [ ] ナビボタンが押しやすい
- [ ] 表が横にはみ出しすぎない
- [ ] 入力欄が見やすい
- [ ] 登録ボタンが押しやすい
- [ ] どこを押せばよいか迷わない

## 8. 次の改善を決める基準

次の Starter Pack は、以下の優先順で決めます。

1. 白画面やエラー
2. 登録や保存ができない場所
3. スマホで使いにくい場所
4. 耳標番号が見にくい場所
5. 操作がややこしい場所
6. 見た目の整理
