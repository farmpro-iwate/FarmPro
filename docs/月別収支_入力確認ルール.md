# 月別収支 入力確認ルール

## 1. 月別収支を正しく出すために大事なこと

月別収支は、出荷・販売管理と経費管理の入力内容から自動集計する。

そのため、以下の入力が特に重要。

## 2. 出荷・販売管理で確認する項目

販売済みの記録では、以下を必ず確認する。

```text
状態：販売済み
販売日：入力あり
販売金額：数字で入力
販売体重：できれば入力
```

## 3. 経費管理で確認する項目

経費記録では、以下を必ず確認する。

```text
支払日：入力あり
経費区分：選択済み
金額：数字で入力
```

## 4. 金額の入力ルール

金額は数字だけで入力する。

良い例:

```text
120000
650000
```

避けたい例:

```text
120,000円
￥120000
六十五万円
```

## 5. 日付の入力ルール

売上は販売日、経費は支払日を基準にする。

```text
売上 → saleDate
経費 → paymentDate
```

## 6. 月がずれる例

販売日が未入力だと、月別売上に入りません。

支払日が未入力だと、月別経費に入りません。

## 7. 確認の流れ

月別収支を見る前に、以下を確認する。

1. 出荷・販売で販売済みデータがあるか
2. 販売日が入っているか
3. 販売金額が入っているか
4. 経費管理で支払日が入っているか
5. 経費金額が入っているか
6. 経費区分が正しいか

## 8. 実運用でのおすすめ

月末にまとめて確認する。

- 出荷・販売の販売済みデータ
- 経費の入力漏れ
- 金額の桁ミス
- 経費区分の間違い
