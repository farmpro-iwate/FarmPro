# Starter Pack 107 入力ルール最終整理

## 1. 目的

FarmProを使いやすくするには、入力ルールを決めておくことが大切です。

入力ルールがないと、あとで検索・集計・履歴確認がしにくくなります。

例:

```text
子牛A
子牛Ａ
A子牛
```

このように表記がバラバラになると、同じ子牛として扱いにくくなります。

---

## 2. 共通ルール

## 2-1. 日付

すべて以下の形にします。

```text
YYYY-MM-DD
```

例:

```text
2026-07-05
```

---

## 2-2. 数字

数字は半角で入力します。

よい例:

```text
1.5
10000
90
```

避ける例:

```text
１．５
１０，０００
九十
```

---

## 2-3. 単位

画面に単位がある場合は、数字だけ入れます。

よい例:

```text
1.5
```

避ける例:

```text
1.5kg
1.5キロ
```

---

## 3. 牛台帳

## 3-1. 必ず入れる項目

```text
名前または耳標番号
生年月日
性別
```

## 3-2. できれば入れる項目

```text
母牛
父牛
メモ
```

## 3-3. 注意点

耳標番号を使う場合は、同じ番号を重複させないようにします。

---

## 4. 子牛管理

## 4-1. 必ず入れる項目

```text
子牛名または耳標番号
生年月日
性別
```

## 4-2. できれば入れる項目

```text
母牛
出生体重
メモ
```

## 4-3. 注意点

給与目安や日齢計算に使うため、**生年月日がとても大事**です。

生年月日がないと、日齢や給与目安が正しく出ません。

---

## 5. 飼料給与実績

## 5-1. 必ず入れる項目

```text
給与日
子牛名
スターター
育成配合
粗飼料
```

## 5-2. 注意点

給与量は数字だけ入力します。

よい例:

```text
0.5
1.2
2
```

避ける例:

```text
0.5kg
少し
多め
```

---

## 6. 給与目安

## 6-1. 必ず入れる項目

```text
日齢
ステージ名
スターター目安
育成配合目安
粗飼料目安
```

## 6-2. 注意点

給与目安は、実際の農場や飼料会社の表に合わせて後で調整します。

今の目安は仮データが含まれるため、実際の表に合わせるのが次の大事な作業です。

---

## 7. 給与アラート対応記録

## 7-1. 必ず入れる項目

```text
対応日
子牛名
アラート種別
対応内容
状態
```

## 7-2. できれば入れる項目

```text
次回確認日
メモ
```

## 7-3. 状態の意味

```text
未対応：まだ確認していない
対応中：対応している途中
対応済み：対応が終わった
様子見：しばらく観察する
再確認必要：もう一度確認が必要
```

## 7-4. おすすめ運用

```text
様子見にしたら次回確認日を入れる
再確認必要にしたら次回確認日を入れる
終わったら対応済みにする
```

---

## 8. 治療記録

## 8-1. 必ず入れる項目

```text
治療日
牛名
症状
処置内容
```

## 8-2. できれば入れる項目

```text
薬剤名
休薬期間
獣医師名
メモ
```

## 8-3. 注意点

薬剤や休薬期間は出荷判断に関わるため、できるだけ正確に入力します。

---

## 9. 出荷・販売

## 9-1. 必ず入れる項目

```text
出荷日または販売日
対象牛
金額
```

## 9-2. できれば入れる項目

```text
出荷先
体重
単価
メモ
```

---

## 10. 経費

## 10-1. 必ず入れる項目

```text
日付
項目
金額
```

## 10-2. 項目例

```text
飼料費
診療費
薬品費
繁殖費
燃料費
資材費
修理費
その他
```

---

## 11. 入力ミスを減らすコツ

```text
毎日まとめて入れすぎない
当日分は当日に入れる
数字は半角で入れる
単位は入れない
名前の表記を統一する
保存後に一覧で確認する
```

---

## 12. 最初のおすすめ運用ルール

まずは難しくしすぎず、以下だけ守れば大丈夫です。

```text
子牛名を統一する
生年月日を必ず入れる
給与実績を日付付きで入れる
対応記録には状態を入れる
様子見・再確認必要には次回確認日を入れる
週に1回バックアップする
```
