# FarmPro 子牛カルテ完成度確認

## 目的

FarmProを完成優先で進めるために、子牛カルテが「子牛情報の終点」として使える状態か確認する。

子牛カルテは、分娩記録から子牛台帳へ登録した後に確認する重要画面。
そのため、単なる基本情報だけでなく、子牛に関係する記録が集まっているかを見る。

## すでに強い点

### 1. 分娩記録からの流れが見える

分娩記録から登録された子牛の場合、子牛カルテ上でその由来が分かる。

これにより、

```text
分娩記録
↓
子牛台帳へ登録
↓
子牛カルテで確認
```

という流れは見えやすくなっている。

### 2. 給与目安が見える

現在の日齢に近い給与目安を表示できる。

確認できる内容:
- 現在日齢
- 近い日齢
- ステージ
- 目標体重
- 目標体高
- 胸囲
- スターター
- 育成配合
- 粗飼料
- その他
- 給与目安メモ

### 3. 給与アラート対応履歴が見える

子牛ごとの給与アラート対応履歴を表示できる。

確認できる内容:
- 対応日
- アラート種別
- 対応内容
- 状態
- 次回確認日
- メモ
- 編集導線

## 足りない可能性があるもの

完成版の子牛カルテとして見ると、以下はまだ集約が弱い可能性がある。

| 区分 | 現状 | 優先度 |
|---|---|---|
| 分娩記録の詳細 | 由来表示はあるが、分娩内容の詳細表示は弱い可能性 | 高 |
| 治療記録 | 子牛カルテ内では未確認 | 高 |
| ワクチン記録 | 子牛カルテ内では未確認 | 高 |
| BLV記録 | 子牛カルテ内では未確認 | 中 |
| 販売・出荷記録 | 子牛カルテ内では未確認 | 中 |
| 予定・次回作業 | 給与アラートの次回確認はあるが、予定全体との連携は未確認 | 中 |

## 次に追加するなら何か

いきなり全部を入れると複雑になる。
まずは、現場で確認頻度が高いものから入れる。

### 優先1: 治療記録

理由:
- 子牛の体調確認に直結する
- 後から見返す価値が高い
- カルテという名前に一番合う

子牛カルテで見たい内容:
- 治療日
- 症状
- 処置内容
- 使用薬
- メモ
- 編集導線

### 優先2: ワクチン記録

理由:
- 子牛ごとに接種履歴を確認したい
- 予定漏れ防止につながる

子牛カルテで見たい内容:
- 接種日
- ワクチン名
- 対象牛
- 次回予定
- メモ

### 優先3: 分娩記録の要約

理由:
- 子牛の出生背景が分かる
- 母牛、実分娩日、出生体重、初乳確認を一緒に見られると便利

子牛カルテで見たい内容:
- 実分娩日
- 母牛耳標番号
- 分娩結果
- 出生体重
- 初乳確認
- 分娩メモ
- 分娩記録への戻り導線

## やらないこと

- 子牛カルテに全記録を一度に詰め込まない
- 専用状態を増やしすぎない
- 入力欄を増やしすぎない
- まずは表示集約を優先する

## Starter Pack 221候補

次は以下のどちらかがよい。

### 候補A: 子牛カルテに治療記録を表示

おすすめ度: 高

理由:
- カルテらしさが一気に上がる
- 子牛の状態確認に直結する
- 完成版に近づく実感が出やすい

### 候補B: 子牛カルテに分娩記録要約を表示

おすすめ度: 高

理由:
- 分娩記録から子牛カルテへの流れがより完成する
- 出生時情報が一画面で分かる

## 結論

現在の子牛カルテは、基本情報・給与目安・給与アラート対応履歴はかなり整っている。

次の完成優先改善は、細かい表示調整ではなく、

```text
子牛カルテに関連記録を集める
```

方向がよい。

最初に入れるなら、**治療記録** または **分娩記録要約** が良い。
