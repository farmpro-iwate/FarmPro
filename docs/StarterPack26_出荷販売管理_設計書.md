# 出荷・販売管理 設計書

## 目的

繁殖Farm Proに、将来的に「出荷・販売管理」を追加するための設計書です。

現在のFarmProには、牛台帳・子牛管理・繁殖・ワクチン・BLV・予定・治療・レポートなどがあります。
次の段階として、販売や出荷の記録を残せるようにすると、経営管理に役立ちます。

---

# 1. 出荷・販売管理でできるようにしたいこと

## 主な目的

- 出荷予定を管理する
- 販売済みの子牛・牛を記録する
- 販売先や市場名を残す
- 販売体重を記録する
- 販売金額を記録する
- 出荷前のワクチン・治療・休薬を確認する
- 月ごとの販売頭数や売上を集計する

---

# 2. 管理対象

## 子牛販売

繁殖和牛農家では、子牛販売の記録が重要です。

記録したい項目:

- 子牛番号
- 子牛名
- 性別
- 生年月日
- 母牛
- 出荷日
- 販売日
- 販売先
- 市場名
- 販売体重
- 販売金額
- 購買者
- メモ

## 成牛販売・淘汰

母牛や成牛の出荷・販売も記録できると便利です。

記録したい項目:

- 個体番号
- 牛名
- 出荷日
- 販売日
- 販売先
- 販売理由
- 販売体重
- 販売金額
- メモ

# 3. 画面案

## 出荷・販売一覧

一覧で確認したい項目:

- 区分
- 対象番号

- 出荷日
- 販売日

- 区分

- 出荷予定日
- 出荷日
- 販売日
- 販売先
- 市場名
- 販売体重
- 販売金額
- 購買者
- メモ
- 状態

## 編集画面

登録済みの出荷・販売記録を修正できるようにします。

## 詳細画面

販売記録の詳細を確認できるようにします。

```text

```

## targetType

対象区分です。

```text
子牛
成牛
その他
```

## status

状態です。

```text
出荷予定
出荷済み
販売済み
取消
```

## saleWeight

販売体重です。
kg単位で、数字だけ入力する想定です。

例:

```text
280
315.5
```

## salePrice

販売金額です。
円単位で、数字だけ入力する想定です。

例:

```text
650000
720000
```

# 5. 既存機能との連携案

## 子牛管理との連携

子牛管理から対象子牛を選べるようにします。

連携したい項目:

- 子牛番号
- 子牛名
- 性別
- 生年月日
- 母牛
- 現在体重

## 牛台帳との連携

成牛を販売する場合は、牛台帳から対象牛を選べるようにします。

連携したい項目:

- 個体番号
- 牛名
- 生年月日
- BLV状態

## 治療管理との連携

出荷前に、休薬期間中でないか確認できると便利です。

確認したい項目:

- 治療中か
- 要再診か
- 休薬終了日が過ぎているか

## ワクチン管理との連携

出荷前に、必要なワクチンや接種状況を確認できると便利です。

## レポートとの連携

将来的には、販売集計をレポートに追加します。

集計したい項目:

- 月別販売頭数
- 月別販売金額
- 平均販売体重
- 平均販売価格
- 子牛販売数
- 成牛販売数

# 6. 保存方法案

現在のFarmProは JSON 保存方式です。

出荷・販売管理を追加する場合、次のファイルを作る想定です。

```text
server/src/data/sales.json
```

API案:

```text
GET    /api/sales
GET    /api/sales/:id
POST   /api/sales
PUT    /api/sales/:id
DELETE /api/sales/:id
```
