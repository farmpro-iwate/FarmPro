# Starter Pack 148 画面表示でIDという言葉を減らす方針

## 1. 目的

画面上の「ID」という表示を減らし、現場で分かりやすい言葉に置き換えます。

---

## 2. 置き換えルール

基本の置き換えは以下です。

```text
母牛ID → 母牛耳標番号
子牛ID → 子牛耳標番号
牛ID → 耳標番号
個体ID → 個体識別番号
```

---

## 3. 残してよいID

以下は、画面に出さない範囲なら残してOKです。

```text
内部データのid
APIのid
JSONのid
プログラム内部のcowId
プログラム内部のcalfId
```

理由は、急に内部名を変えるとアプリが壊れるからです。

---

## 4. 画面では避けたい言葉

画面では、以下のような言葉はなるべく避けます。

```text
ID
cowId
calfId
recordId
```

---

## 5. 画面で使いたい言葉

画面では、以下を使います。

```text
耳標番号
母牛耳標番号
子牛耳標番号
個体識別番号
```

---

## 6. メモの活用

まだ個体識別番号の専用欄がない画面では、メモに書いてもOKです。

例:

```text
個体識別番号: 1234567890
```

---

## 7. 今後の修正対象

今後、以下の画面を順番に見直します。

```text
子牛台帳
牛台帳
出荷・販売
治療管理
ワクチン管理
BLV管理
レポート
```

---

## 8. 注意

全部を一気に変えません。

理由:

```text
既存データが壊れる可能性がある
画面が白くなる可能性がある
どのファイルで使っているか確認が必要
```

そのため、1画面ずつ安全に進めます。
