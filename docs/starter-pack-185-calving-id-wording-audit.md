# Starter Pack 185: 分娩まわりのID文言監査

## 目的

分娩記録まわりで、画面やCSVに raw ID を直接見せない方針を整理する。
現場では耳標番号を中心に確認し、内部的なIDはできるだけ表示しない。

## 184までに整理した箇所

- 分娩記録一覧の案内文から `子牛ID` 表現を削除
- 分娩記録一覧のCSV列名を `子牛カルテ直接リンク` に変更
- CSVには `calfId` の値を直接出さず、直接リンクがある場合だけ `あり` と出す
- 分娩記録編集画面の登録状況で `calfId` を直接表示しない
- 分娩記録編集画面の `子牛IDがないため` を `直接リンク情報がないため` に変更
- 分娩記録編集画面の `分娩記録IDがありません` を、一覧から開き直す案内に変更

## 検索確認

`子牛ID`、`分娩記録ID`、`calfId` を検索し、分娩まわりのユーザー向け表示として残っていないか確認した。
少なくとも今回の対象フローでは、raw ID を直接見せる文言は整理済み。

## 今後の方針

- 画面表示は `耳標番号` を優先する
- 内部リンクの有無は `直接リンク情報` または `子牛カルテへ直接移動できます` と表現する
- CSVでも raw ID ではなく、業務上の意味が分かる状態名にする
- どうしても内部IDが必要な場合は、開発者向けログやデバッグ用に限定する
